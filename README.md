# SDD — Spec Driven Development

> La especificación precede al código

Plugin de desarrollo dirigido por especificación. Cierra la brecha entre la intención y el código con verificación de integridad en cada paso.

---

## 🚀 Instalación Rápida

```bash
git clone https://github.com/JaviMontano/mao-sdd.git ~/.claude/plugins/mao-sdd
```

- IIKit integrado (Intent-Integrity Chain)
- Dashboard interactivo de estado
- Verificación de integridad en cada paso
- Multi-plataforma (Claude/Codex/Gemini/OpenCode)
- Trazabilidad completa de decisiones

## 📐 Arquitectura

specify → plan → checklist → testify → tasks → analyze → implement + dashboard interactivo

# SDD — Spec Driven Development

> Specification precedes code

Specification-driven development plugin. Closes the gap between intent and code with integrity verification at each step.

### What It Does

- specify→plan→checklist→test→tasks→analyze→implement pipeline
- Built-in IIKit (Intent-Integrity Chain)
- Interactive status dashboard
- Integrity verification at each step
- Multi-platform (Claude/Codex/Gemini/OpenCode)
- Full decision traceability

### Architecture

```
specify → plan → checklist → testify → tasks → analyze → implement + interactive dashboard
```
